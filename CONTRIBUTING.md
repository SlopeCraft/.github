# Contributing to SlopeCraft

Thank you for contributing to a repository in the SlopeCraft organization.

This document is the organization-wide default. A repository-specific contributing guide takes precedence when present.

## Before opening an issue

- Search existing issues and discussions first.
- Use issues for reproducible defects, repository-scoped feature requests, and maintainership work.
- Use [SlopeCraft Discussions](https://github.com/SlopeCraft/SlopeCraft/discussions) for product usage questions and general troubleshooting.
- Do not disclose security vulnerabilities in public issues. Follow the [security policy](https://github.com/SlopeCraft/.github/security/policy).

## Opening a useful issue

Please include enough information for another person to understand and reproduce the problem:

- the affected repository, version, tag, or commit;
- the operating system, toolchain, runtime, or relevant environment;
- minimal reproduction steps;
- expected and actual behavior;
- logs, screenshots, or sample files with sensitive information removed.

Keep one issue focused on one problem or one coherent feature.

## Pull requests

Before starting a large or compatibility-sensitive change, open an issue or discussion to confirm scope.

A pull request should:

- have a clear purpose and limited scope;
- explain the motivation and user-visible effect;
- include appropriate tests or validation;
- update documentation when behavior, interfaces, formats, or workflows change;
- preserve compatibility unless the break is intentional and documented;
- avoid unrelated formatting or refactoring;
- identify third-party code, data, assets, and their licenses.

Both Chinese and English are welcome. Keep terminology and identifiers consistent with the target repository.

## Licensing

By contributing, you agree that your contribution may be distributed under the license of the target repository.

Do not assume that one SlopeCraft repository's license applies to another repository. Verify the target repository's license before contributing. Do not submit material that you do not have the right to redistribute.

## Review and maintenance

Maintainers may request changes, split an oversized contribution, or decline work that does not fit the repository's scope or maintenance capacity.

# 参与 SlopeCraft 贡献

感谢你参与 SlopeCraft 组织旗下仓库的开发与维护。

本文档是组织级默认贡献指南。如果目标仓库提供了自己的贡献指南，应以仓库内的说明为准。

## 提交 Issue 前

- 先搜索已有 Issue 和 Discussion。
- Issue 适用于可重现的缺陷、限定于当前仓库的功能请求，以及维护事务。
- 产品使用问题和一般故障排查请前往 [SlopeCraft Discussions](https://github.com/SlopeCraft/SlopeCraft/discussions)。
- 不要在公开 Issue 中披露安全漏洞，请遵循[安全策略](https://github.com/SlopeCraft/.github/security/policy)。

## 提交高质量 Issue

请提供足够的信息，使其他人能够理解并重现问题：

- 受影响的仓库、版本、标签或提交；
- 操作系统、工具链、运行时或其他相关环境；
- 最小重现步骤；
- 预期行为与实际行为；
- 已移除敏感信息的日志、截图或示例文件。

一个 Issue 应只聚焦一个问题或一组紧密相关的需求。

## Pull Request

开始大型改动或可能影响兼容性的改动前，建议先通过 Issue 或 Discussion 确认范围。

Pull Request 应当：

- 目标明确、范围有限；
- 说明改动动机和用户可见影响；
- 包含适当的测试或验证；
- 在行为、接口、格式或工作流变化时同步更新文档；
- 默认保持兼容，破坏性变化必须有明确理由和迁移说明；
- 避免无关的格式化或重构；
- 标明第三方代码、数据、素材及其许可证。

中文和英文均可使用。请与目标仓库现有术语和标识符保持一致。

## 许可证

提交贡献即表示你同意该贡献可按照目标仓库的许可证分发。

不要假设某个 SlopeCraft 仓库的许可证自动适用于其他仓库。提交前请核验目标仓库的许可证，也不要提交无权再分发的内容。

## 审查与维护

维护者可以要求修改、拆分范围过大的贡献，或因不符合仓库职责及维护能力而拒绝合并。
